MochaKde

A reproducible, gaming-oriented NixOS/KDE system from Diesel OS Lab. MochaKde focuses on performance, visual polish, safe rebuild discipline, local cache strategy, and public credit for the upstream work that makes it possible.

Mission

MochaKde exists to make a high-performance Linux gaming system reproducible, understandable, and accessible.

Accessible performance

The project studies how to get strong gaming performance without depending on expensive licenses, paid tuning tools, or unnecessary hardware upgrades.

Reproducible systems

Working states are documented with manifests, logs, cache paths, pinned inputs, and bootable generations instead of one-off manual tweaks.

Social purpose

Diesel OS Lab has a nonprofit/beneficial purpose, including support for social and educational initiatives that help children with Down syndrome and their families access technology, learning resources, digital inclusion, and support programs.

Why KDE Plasma?

MochaKde is the KDE/Plasma branch of the Mocha project. It is intentionally kept separate from the GNOME edition to avoid mixing desktop-specific assets, services, themes, and assumptions.

Fast, flexible desktop

KDE Plasma gives MochaKde a highly configurable desktop suitable for a familiar, gaming-friendly layout, while still allowing deep control over visuals and workflow.

Separate recipe

MochaKde has its own repository and KDE-specific modules. The GNOME Mocha system remains separate, so experiments in Plasma do not pollute the stable GNOME recipe.

Technical principles

MochaKde favors careful validation over risky changes. Kernel, NVIDIA, desktop, theme, and gaming changes are treated as reproducibility-sensitive work.

Safe rebuild flow

The default flow is dry-build, build, then boot. Live system switching is not the default path for risky kernel, NVIDIA, KDE, or desktop changes.

Kernel/NVIDIA pairing

MochaKde studies pinned kernel and NVIDIA combinations, including CachyOS-derived kernel work and explicit driver pairings for repeatable gaming tests.

Local cache discipline

MOCHAFAST is used for logs, snapshots, manifests, study mirrors, and cache strategy. Good states are recorded before they become canonical.

MochaKde does not import raw Arch/CachyOS packages directly into NixOS. The safe path is Nix/NixOS packaging, pinned inputs, explicit manifests, and bootable generations.

Upstream credits

MochaKde is an integration and validation project. It depends on major upstream work and credits that work explicitly.

Yuhui Xu

MochaKde uses and studies the community Nix project xddxdd/nix-cachyos-kernel, maintained by Yuhui Xu. This project provides CachyOS kernel variants as Nix/NixOS packages with support for reproducible integration.

CachyOS Team and contributors

The upstream CachyOS kernel patchset, tuning work, and kernel variants come from the CachyOS project and its contributors. MochaKde does not claim authorship of that upstream work.

License clarification for xddxdd/nix-cachyos-kernel was requested in issue #74. Yuhui Xu replied that GPLv2 was added, the stated use cases are allowed, and the correct credit name is “Yuhui Xu”.

Project status

MochaKde is under active development. The current focus is preserving good KDE visual states, validating CachyOS/Caninana kernel recipes, pinning NVIDIA behavior, and recording every working state so it can be reproduced later.

Current priority

Keep the KDE recipe separate, beautiful, reproducible, and safe to rebuild.

Known-good philosophy

A state only becomes canonical after it is tested, logged, documented, and confirmed.

Public responsibility

Credits, licenses, upstream references, and derivative work boundaries are documented before the project presents them publicly.
